Assignment 2 - Designing Apps for Testing (15%) 
Ankitraj Kaur(742944) 
 
 
1.Consider the Android PONG game that we built in class. Is the game designed to be 
tested using manual or automated processes? 
-If manual, explain why. In your answer, provide examples. 
-If automated, explain why. In your answer, provide examples. 
Ans- According to me, PONG game is designed to be tested using manual processes 
because in this game we have less features and automatic testing is used to test 
repeated features but in this game main feature is to hit the ball with a racket and when 
the racket hits the ball score increased which can easily tested with manual testing. 
 
 
2.Suppose you are creating a test plan for the PONG game. Describe 10 features that you 
would need to test. For each feature, describe: 
Features and behaviour 
F1.Feature - API version of the screen 
Expected Behavior-API version should be more than 24 to install the PONG game. 
 
F2.Feature-Initial position of the ball 
Expected Behaviour-Initial position of ball should be at the center of the screen. 
 
F3.Feature-Initial position of the Racket 
Expected Behaviour-Initial position of the racket should be at the center of the screen  
but below the ball. 
 
F4.Feature-Initial Score 
Expected Behaviour- Initial score should be zero. 
 
F5.Feature-Score after racket hits the ball 
Expected Behaviour-Score should increase when racket hits the ball. 
 
F6.Feature-Score when racket miss the ball. 
Expected Behaviour-Score decreases when racket miss the ball. 
 
F7.Feature -Movement of ball 
Expected Behavior-Ball moves up and down  
 
F8.Feature-Ball boundaries 
Expected Behavior- Ball should bounce back when it collides with the top screen. 
 
F9. Feature-Movement of racket 
Expected Behaviour-Racket moves left and right on mouse click 
 
F10.Feature-Lives when game starts 
Expected Behavior-Initial lives should be 3
